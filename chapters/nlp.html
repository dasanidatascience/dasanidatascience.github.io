I. Introduction¶

Data scientists frequently split data into two categories: structured and unstructured data. The former is the traditional kind of data used by engineers because it fits neatly into the row and column databases used by so many websites and softwares. However, unstructured data is far more messy and difficult for a machine to store, analyze, and interpret. The data generated from coversation, articles, and social media can carry lots of useful information, but these are all examples of unstructured data, so they are difficult to manipulate. However, recent advances in machine learning have allowed us to not only take advantage of this data, but also build models that can understand the meaning behind the words in the texts themselves.

Natural language processing (NLP) is a field of machine learning that gives the machines the ability to read, understand and derive meaning from human language. There are many use cases for NLP, some of which you may even use on a daily basis! Siri, Alexa, and other digital assistants rely on NLP models to understand the requests posed to them, and to formulate responses in a way that is natural for human language. On the corporate level, businesses can take advantage of NLP algorithms to analyze social media data to understand the sentiment consumers have about their products.

In this case study, we will explore major concepts in NLP through analyzing user tweets, and then build a model to automatically generate tweets in the likeness of an existing account!

II. Invoking APIs¶

There are two primary ways we can acquire data from the internet. The first one is web scraping, which we explored in great detail in this notebook. We rely on web scraping when a site does not provide a way for us to directly acquire their data, so we have to scrape it off their webpage instead.

However, sometimes a site will provide a method to programmatically access and utilize its data in the form on an API. An Application Programming Interface (API) is a method in which different servers can talk to one another using requests. For example, if you go on Google and search "weather in houston", Google sends a request to weather.com's API asking for weather information in Houston, TX. The API processes this request internally, and returns weather data to Google, who displays it on their search page for you to view.

But why would companies just provide their data externally? Typically, this data and API service are packaged as a product, and those who query an API are usually charged for the service. For this reason, when you call an API, you often have to authenticate yourself with some kind of unique token; this key lets the API know who you are, much like a username and password on a user-facing website.

To generate the data for this case study, we will make use of the Twitter API to retrieve tweets!

  1. curl/requests: curl is the commmand to make a request to an API, while requests is the python package built for this purpose.
  2. GET/get(): This is specifying the request method, in this case a GET request. We use this kinds of request method when we'd like to retrieve information from the server. There are other request methods as well. For example, a POST request is used to send data to the server.

In this case we used parameters screen_name and count with values elonmusk and 1, respectively. def clean_tweet(tweet):
    
    tweet = re.sub(r'http\S+', '', tweet)           #Remove URLs
    tweet = re.sub(r'@[a-zA-Z0-9_]+', '', tweet)    #Remove @ mentions
    tweet = tweet.strip(" ")                        #Remove whitespace resulting from above
    tweet = re.sub(r' +', ' ', tweet)               #Remove redundant spaces
    tweet = re.sub('#', '', tweet)                  #Remove '#' hash tag
    tweet = re.sub('RT[\s]+', '', tweet)            #Remove RT (retweet)

    # Handle common HTML entities
    tweet = re.sub(r'&lt;', '<', tweet)
    tweet = re.sub(r'&gt;', '>', tweet)
    tweet = re.sub(r'&amp;', '&', tweet)
    
    return tweet

Now that we have cleaned the tweets, we can use a package called TextBlob to understand some aspects of a tweet's sentiment. First we will look into polarity, which quantifies the emotion of a tweet. The metric is a value in the range of $[-1.0, 1.0]$, where positive and negative values denote positive and negative sentiment, respectively, and the absolute value denotes the magnitude of the emotion. Another sentiment we can observe is subjectivity, how factual or opinionated a tweet is. This metric is a value in the range of $[0.0, 1.0]$; generally, the lower the value, the more objective the statement.

As you can see below, for us getting these sentiments is as easy as a single line of code. But how is TextBlob computing these values? Behind the scenes, this package uses a trained Recurrent Neural Network (RNN) which takes the words in these tweets as input and predicts their sentiments by attempting to understand the meaning of the text. We will dive into the details of RNNs in the next section.

def get_subjectivity(tweet):
    return TextBlob(tweet).sentiment.subjectivity

def get_polarity(tweet):
    return TextBlob(tweet).sentiment.polarity

def get_sentiment(score):
    if score < -0.1:
        return 'Negative'
    elif score > 0.1:
        return 'Positive'
    else:
        return 'Neutral'

IV. Recurrent Neural Networks¶

Traditional neural networks are proficient in completing tasks where the input features are semi-independent from one another. For example, predicting a person's height based on their gender and age is a task this algorithm can sufficiently complete, because gender and age are independent of one another. However, for problems with sequential inputs, such as text, this assumption is invalid because any given word is dependent on the ones before it.

This is where RNN's come into play. Recurrent neural networks (RNNs) are a class of neural networks that allow previous outputs to be used as inputs while having hidden states. RNNs are called recurrent because they perform the same task for every element of a sequence, with the output being depended on the previous computations. Another way to think about RNNs is that they have a memory which captures information about what has been calculated so far. The figure below illustrates the architecture of a simple RNN.



The left side is traditional RNN notation, but it is hard to understand what exactly is taking place. The right side shows the RNN being "unfurled", so that the network's state for each step in the sequence is revealed. For example, if we had a sequence of three words, we would unroll this RNN into three layers, one for each word. Let's break this network down piece by piece.

$x(t)$: This is the input for the network at time step $t$. In the case of the three word input sequence, $x(1)$ would be the first word, $x(2)$ would be the second, and so on.

$h(t)$: This represents the hidden state of the network at time $t$. The hidden state is a trademark of the RNN, and is vital since it acts as the "memory" of the network. The hidden state at time $t$ is computed as,

$h(t) = f(\begin{bmatrix}b & U & W \end{bmatrix} \begin{bmatrix}1 \\ x(t) \\ h(t-1) \end{bmatrix})$

where $f$ is a non-linear transformation, such as the tanh or reLU functions, and $b$, $U$ and $W$ are weights that will be tuned when training the algorithm (note that $h(0)$ is typically defined as a random integer between $0$ and $1$).

$o(t)$: This is the output of the network at time $t$. This value is computed as,

$o(t) = \begin{bmatrix}c & V \end{bmatrix} \begin{bmatrix}1 \\ h(t) \end{bmatrix}$

where $c$, and $V$ are weights that will be tuned when training the algorithm. Some implementations may also pass all outputs through some non-linear transformation, or just the final output.

Let's put it all together with a simple example. Suppose I have a vocabulary consisting of only the words ["ate", "cat", "food", "the"]. I provide my network this vocabulary along with the text sequence "the cat ate". at timestep $t$, the network takes as input word $t$ in the text sequence and outputs a probability score for each word in the vocabulary. For example, at timestep 1, the RNN will receive "the" as input and guess the likelihood for each word in the vocabulary to come next. Luckily, in the next time step it is given the next word, so it can train itself to guess these probabilities in an intelligent way. This process will repeat itself until it gets to the last word, "ate", and predicts probability scores for each word in the vocabulary. We can then take the one with the highest score and use it as our output. You can then repeat this process by passing the network it's own guess as an input and have it generate another word. This is a simplified explanation of how an RNN can generate text.

Now we can train an RNN to generate tweets. textgenrnn is a python package that is fantastic for this purpose, since it will build and train a model according to our specifications.

First, we configure the architecture of the model. RNN's can be used to generate text on either a character level or a word level. Word level models require a lot more data than their counterparts, and therefore take much longer to train. We also need to configure the size of the RNN's hidden state embedding, and the number of layers in the RNN. If we are limited on training data we can allow our RNN to predict text forwards and backwards to increase the number of examples; however, this is discouraged if we can just generate more training data (which we can by scraping more tweets). Finally, we can set the length of the input sequence, and restrict the size of the vocabulary if our RNN is a word lebel model.

Once the model is fully specified, we need to configure training parameters. The number of epochs signifies how many times the model will be trained with the full dataset. The larger the number of epochs, the more accurate the model but also the longer it will take to train. We can also configure the training to validation ratio, dropout, and validation parameters to increase the model's ability to generalize.

Once we have fully specified the architecture of the model and training procedure, we can begin to train the model!

def train_model(model_name, file_name=model_name, rnn_size=128, rnn_layers=3, num_epochs=20,batch_size=1024,dim_embeddings=100):
    file_name = file_name + '.txt'

    model_cfg = {
      'word_level': False,          # set to True if want to train a word-level model (more data/ smaller max_length)
      'rnn_size': rnn_size,         # number of LSTM cells of each layer (128/256 recommended)
      'rnn_layers': rnn_layers,     # number of LSTM layers (>=2 recommended)
      'rnn_bidirectional': False,   # consider text both forwards and backward, can give a training boost
      'max_length': 30,             # number tokens to consider before predicting\(20-40 for characters, 5-10 for words)
      'max_words': 10000,           # maximum number of words to model; the rest will be ignored (word-level model only)
      }

    train_cfg = {
      'line_delimited': False,      # set to True if each text has its own line in the source file
      'num_epochs': num_epochs,     # set higher to train the model for longer
      'gen_epochs': 5,              # generates sample text from model after given number of epochs
      'train_size': 0.8,            # proportion of input data to train on: setting < 1.0 limits model from learning perfectly
      'dropout': 0.0,               # ignore random proportion of source tokens each epoch, allowing model to generalize better
      'validation': False,          # If train__size < 1.0, test on holdout dataset; will make overall training slower
      'is_csv': False               # set to True if file is a CSV exported from Excel/BigQuery/pandas
      }

    textgen = textgenrnn(name=model_name)

    #train model using tweets file and prespecified parameters
    train_function = textgen.train_from_file if train_cfg['line_delimited'] else textgen.train_from_largetext_file
    train_function(
      file_path=file_name,

Now we can use the trained model to generate tweets! One issue commonly faced with text generation is that conservative models may output the same or similar text repetitively because the model because those words are ones the model is more confident about. For example,

is that they were all the same thing that was a startup is that they were all the same thing that was a startup is that they were all the same thing that was a startup is that they were all the same

To alleviate this issue we introduce the temperature hyperparameter; higher values of this parameter allow the model to be more diverse with its vocabulary choice at the risk of making more mistakes. Like most other things in machine learning, setting the correct temperature value is a tradeoff between accuracy and diversity. Generally, a value between $0.1$ and $1.0$ is recommended.

def generate_tweets(username, count, n=5, temperature=0.2):
    model_name = username + '_' + str(count)

    #train model if unable to locate weight, vocab, and config files
    if not (path.exists('{}_weights.hdf5'.format(model_name)) 
          and path.exists('{}_vocab.json'.format(model_name)) 
          and path.exists('{}_config.json'.format(model_name))):
    write_to_file(username, count)
    train_model(model_name)

    #load model and generate tweets
    textgen = textgenrnn(weights_path='{}_weights.hdf5'.format(model_name),
                      vocab_path='{}_vocab.json'.format(model_name),
                      config_path='{}_config.json'.format(model_name))
    tweets = textgen.generate(n=n, temperature=temperature, return_as_list=True)

    return tweets
